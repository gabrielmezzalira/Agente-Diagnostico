## Spacing Scale

All tokens are multiples of 4. Declared values:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status dot margins |
| sm | 8px | Compact element spacing, input internal padding (vertical) |
| md | 16px | Default element spacing, column internal padding, card padding |
| lg | 24px | Section padding within form, card grid gap |
| xl | 32px | Form section breaks |
| 2xl | 48px | Page-level top padding |
| 3xl | 64px | Major layout gaps (not used in Phase 1) |

Exceptions:
- Topbar height: 52px (established now for Phase 6 reference — not a spacing token, structural)
- Budget bar height: 36px (structural)
- Button height: 34px (standard action buttons)
- Pin/dismiss buttons: 26×26px (touch target minimum for inline card actions)
- Status dot: 7px diameter (decorative, not a spacing token)
- Coverage status dot: 8px diameter
- TTL bar: 2px height (Phase 7, established in Tailwind config now)
- Budget fill bar: 6px height (Phase 8, established in Tailwind config now)
- Mini progress bar (coverage card): 3px height

## Typography

| Role | Size | Weight | Font | Line Height | Letter Spacing | Usage |
|------|------|--------|------|-------------|----------------|-------|
| Label uppercase | 10px | 600 | DM Sans | 1.2 | 0.1em | Section headers, block tags, status labels, field group titles |
| Small | 12px | 400 | DM Sans | 1.4 | normal | Helper text, timestamps, secondary metadata |
| Small mono | 12px | 400 | DM Mono | 1.4 | normal | Budget values (secondary), cost figures |
| Body | 13px | 400 | DM Sans | 1.5 | normal | Form field values, card body text, placeholder text |

| Mono primary | 13px | 400 | DM Mono | 1.4 | normal | Session timer display |

Rules:
- Nothing exceeds 14px in the application UI (source: CONTEXT.md — "section headings: nothing above 14px")
- DM Mono is exclusively for: timer, budget consumed/limit values, token counts, shortcut badges inside buttons
- All uppercase labels use `font-size: 10px; font-weight: 600; letter-spacing: 0.1em; text-transform: uppercase`
- No italics in Phase 1 components

### Accent (`#22a267`) Reserved For

The green accent is used ONLY on:
1. Primary action buttons (Gerar Perguntas — background fill)
2. Active session "ao vivo" badge — background or dot color
3. Selected state on project type selector cards (border highlight)
4. DMS slider active track and thumb
5. Queued question card border and background tint (via `green-bg-light`)
6. Pinned question card border
7. Block tags (bg `#edfaf3`, text `#22a267`)
8. Pin button hover state
9. Budget fill bar when usage < 60%
10. Coverage status dot — "covered" state

Accent is NOT used on: navigation links, general hover states, text links, form focus rings (use `border-hover` `#cccccc` instead), or body text.

Full monitoring components are specified here for Tailwind token establishment but implemented in Phase 6+.

```
bg: #ffffff
border: 1px solid #e8e6e0
border-radius: 10px
padding: 16px
hover: border-color #cccccc, box-shadow: 0 2px 8px rgba(0,0,0,0.06)
```
