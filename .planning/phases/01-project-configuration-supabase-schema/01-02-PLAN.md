Supabase Vault call pattern (from RESEARCH.md Pattern 1):
- Create: `SELECT vault.create_secret($key, 'gemini_key_project_' || $name, 'Gemini API key') AS id`
  → returns UUID; store in projects.gemini_api_key_secret_id
- Update (key changed): `SELECT vault.update_secret($existing_uuid, $new_key)`
  → UUID stays the same in projects row

asyncio.to_thread pattern (MANDATORY for all supabase-py calls):

result = await asyncio.to_thread(
